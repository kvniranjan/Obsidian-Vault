## What It Does
StoryMachine is a Python CLI tool that converts Product Requirements Documents (PRDs) and Technical Specifications into structured, context-enriched user stories with acceptance criteria. It reads your repository via GitHub/GitLab APIs to understand the codebase context, then uses AI (OpenAI) to break work down into right-sized user stories designed to be consumed by both human developers and AI coding agents.

## Key Features
- Converts PRD + Tech Spec markdown files into structured user stories with acceptance criteria
- Reads repository context via GitHub/GitLab integration for code-aware story generation
- CLI-based workflow that fits into existing development pipelines
- Outputs stories in a format optimized for AI agent consumption (Claude, Copilot, etc.)
- Built on the "unit of work" philosophy — right-sized tasks that reduce AI error compounding

## Installation
```bash
git clone https://github.com/nilenso/storymachine.git
cd storymachine
uv sync
```
Create a `.env` file with:
```
OPENAI_API_KEY=your-key
GITHUB_TOKEN=your-github-pat   # repo scope
GITLAB_TOKEN=your-gitlab-pat   # read_api, read_repository scopes
```
Run with:
```bash
uv run storymachine --prd path/to/prd.md --tech-spec path/to/spec.md --repo https://github.com/owner/repo
```
