## How to Use It in Your BA Workflow (Obsidian)

### 1. Generate BRD/PRD Drafts and Import into Obsidian
Use Specifai to input a project idea (e.g., "Core banking loan origination module migration") and let it auto-generate a full BRD and PRD. Export the generated documents as markdown, then copy them into your Obsidian vault under a `Requirements/` folder. Add `[[Project]]` links and frontmatter tags so they integrate with your existing note graph. This replaces hours of blank-page drafting with a structured first draft you can refine.

### 2. Convert Meeting Notes into Structured Requirements
After a stakeholder meeting, paste your Granola meeting transcript into Specifai's chat interface with a prompt like: *"Extract all functional requirements from this meeting transcript and format as a PRD."* Specifai will parse the conversation and produce structured requirements with IDs. Save the output as an Obsidian note linked to the original meeting note using `[[2026-03-19 Stakeholder Call]]` backlinks.

### 3. Auto-Generate User Stories for Sprint Planning
Once your BRD is finalized in Specifai, use its story generation feature to break requirements into user stories with acceptance criteria. Export these as individual markdown files (one per story) into an Obsidian `User Stories/` folder. Use Obsidian's Dataview plugin to create a dashboard: `TABLE status, priority FROM "User Stories" WHERE project = "Loan Migration"` — giving you a live sprint board inside your vault.

### 4. Sync Requirements to Jira and Track in Obsidian
Use Specifai's Jira integration to push generated stories directly into your Jira project. Then maintain a mapping note in Obsidian like `Jira Sync Log.md` that tracks which Obsidian requirement notes correspond to which Jira tickets (e.g., `[[REQ-001 Loan Eligibility]] → JIRA-4521`). This gives you a single source of truth in your vault while keeping the dev team's Jira board in sync.

### 5. Generate Test Cases for Gap Analysis
Use Specifai's test case generation to automatically create test scenarios from your user stories. Export these into an Obsidian `Test Cases/` folder and cross-link them to requirements: `[[REQ-001]] → [[TC-001 Verify Eligibility Rules]]`. Use this traceability to run gap analysis — any requirement without a linked test case is a gap. Create a Dataview query: `LIST FROM "Requirements" WHERE !contains(file.outlinks, "TC-")` to surface untested requirements instantly.
