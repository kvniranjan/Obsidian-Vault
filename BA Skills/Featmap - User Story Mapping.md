# Featmap - User Story Mapping

## What is it?
Featmap is a simple, open-source user story mapping tool built to help agile teams organize user activities, user tasks, and user stories in a visual two-dimensional map. It can be self-hosted or used as a cloud service. Available at [github.com/amborle/featmap](https://github.com/amborle/featmap).

## Why it matters for Business Analysts
User Story Mapping (popularized by Jeff Patton) is a core BA/product technique for organizing requirements around the user journey rather than a flat backlog. Featmap provides a free, dedicated tool for this — avoiding the complexity of general-purpose tools like Jira or Miro while keeping story maps visual, shareable, and structured. It's particularly useful when working with agile teams to prioritize releases and identify MVP scope.

## How to use it in BA Workflows
1. **Discovery to Backlog** — After stakeholder interviews, organize discovered needs into the story map: activities (top level) → tasks (middle) → user stories (bottom).
2. **Release Planning** — Draw horizontal slices across the map to define release boundaries — what goes into MVP vs. later iterations.
3. **Gap Analysis** — Visual representation makes it easy to spot missing workflow steps or user tasks that haven't been covered.
4. **Stakeholder Alignment** — Share the story map link with stakeholders to validate scope and priority in sprint planning sessions.
5. **Backlog Generation** — Export or screen-capture story maps as input to Jira/Azure DevOps when creating the formal product backlog.

## Key Features
- Two-dimensional story map layout: Activities → Tasks → Stories
- Drag-and-drop card organization
- Release slicing — define MVP and future release boundaries visually
- Collaborative — multiple users can work simultaneously
- Self-hostable (Go backend, open-source under MIT license)
- Lightweight and purpose-built (no feature bloat vs. general tools)
