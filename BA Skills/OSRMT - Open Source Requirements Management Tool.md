# OSRMT - Open Source Requirements Management Tool

## What is it?
OSRMT (Open Source Requirements Management Tool) is a free, configurable, platform-independent requirements management application for defining, organizing, and tracing software requirements across the full development lifecycle. It supports single-user desktop usage or multi-user deployment via a centralized web server.

**GitHub:** [osrmt/osrmt](https://github.com/osrmt/osrmt)

## Why it matters for Business Analysts
Requirements management is at the heart of BA work. OSRMT provides a structured environment to capture and trace requirements from initial feature definition through to test cases — ensuring nothing falls through the cracks. It replaces sprawling spreadsheets with a purpose-built tool that enforces traceability and version control. Key BA benefits:
- Ensures requirements are linked to business features, use cases, design, and test cases
- Supports change control — critical when requirements evolve mid-project
- Enables audit trails for compliance-heavy domains (banking, healthcare, government)
- Multi-user collaboration allows BAs, developers, and testers to work from the same source of truth

## How to use it in BA Workflows

### 1. Define Features → Requirements
Capture high-level business features, then decompose them into detailed functional and non-functional requirements within OSRMT's hierarchy.

### 2. Link Use Cases to Requirements
Attach use cases to requirements so stakeholders can see the full context behind each specification.

### 3. Establish Traceability Matrix
Use OSRMT's built-in traceability to map: Feature → Requirement → Design → Implementation → Test Case. This is the RTM (Requirements Traceability Matrix) — a core BA deliverable.

### 4. Manage Change Control
When a requirement changes, OSRMT tracks the version history and lets you assess downstream impacts before accepting changes.

### 5. Generate Reports
Use the custom report engine to produce stakeholder-friendly exports showing requirement status, coverage gaps, and traceability chains.

## Key Features
- Full requirements lifecycle: Features, Requirements, Use Cases, Design, Implementation, Test Cases
- Built-in Requirements Traceability Matrix (RTM)
- Version control and change control module
- Hierarchical requirement organization (parent-child linking)
- Custom configurable workflows
- Multi-user web application or single-user desktop mode
- Cross-platform: Windows, Mac, Linux
- Database support: MySQL, PostgreSQL, Oracle Express, MS Access
- Phase-based work planning
