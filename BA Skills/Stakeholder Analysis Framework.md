# Stakeholder Analysis Framework

## What is it?
The Stakeholder Analysis Framework is a structured set of templates and methodologies for identifying, categorizing, and managing project stakeholders. The GitHub repository [joelparkerhenderson/stakeholder-analysis](https://github.com/joelparkerhenderson/stakeholder-analysis) provides a comprehensive markdown-based template covering the Power-Interest Grid, stakeholder registers, communication plans, and engagement strategies. A companion webapp at [papanito/stakeholder-analysis](https://github.com/papanito/stakeholder-analysis) provides a graphical power-interest grid visualization tool.

## Why it matters for Business Analysts
Stakeholder analysis is a foundational BA activity — identifying the right stakeholders early prevents requirement gaps and miscommunication later. The Power-Interest Grid helps prioritize engagement effort: high power/high interest stakeholders need active management, while low power/low interest stakeholders need minimal monitoring. Structured templates ensure consistency and repeatability across projects.

## How to use it in BA Workflows
1. **Project Kickoff** — Use the stakeholder register template to list all stakeholders identified from organizational charts, project briefs, and sponsor conversations.
2. **Power-Interest Mapping** — Plot each stakeholder on the 2x2 grid (Power vs. Interest) to determine engagement strategy:
   - High Power, High Interest → Manage Closely
   - High Power, Low Interest → Keep Satisfied
   - Low Power, High Interest → Keep Informed
   - Low Power, Low Interest → Monitor
3. **Communication Planning** — For each stakeholder quadrant, define communication frequency, channel (email, meeting, dashboard), and message type.
4. **Ongoing Tracking** — Update the register as stakeholders' roles or interest levels shift during the project lifecycle.
5. **RACI Alignment** — Cross-reference stakeholder analysis with a RACI matrix to confirm decision rights and accountability.

## Key Features
- Power-Interest Grid template (markdown and visual webapp versions)
- Stakeholder register structure: name, role, influence level, interest area, communication preference
- Engagement strategy guidance by quadrant
- Open-source templates adaptable to any project or domain
- Graphical webapp for interactive grid positioning (papanito/stakeholder-analysis)
- BABOK-aligned stakeholder identification approach
